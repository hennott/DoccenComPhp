Das Grabber-Modul kopiert entsprechend der Konfiguration Dateien aus dem SourceS-Ordner in die entsprechenden Order der Dokumentation. Das ist dann nötig, wenn Sie mittels Skript alle Dateien dort hinkopieren und daraus verschiedene Dokumentationen ableiten. Dies entspricht dem zweiten Schritt im Workflow-Modul.

Dieses Modul kann auch durch externe Tools ersetzt werden, indem man einen SyncJob entsprechend konfiguriert.

Modus "Add"

In diesem Modus kann man Fragmente (PartS) einer Zieldatei zuordnen, die im SourceS Verzeichnis liegt.

Modus "Remove"

Im Remove-Modus werden die Dateien NICHT gelöscht, sondern deren Zuordnung zu einer Zieldatei wird aufgehoben.

Modus "Run"

Der Modus Run führt alle Vorgänge für das Zusammenführen aus.

PDFe Erzeugung

PDFe zur Erzeugung von PDF-Dateien

Für die Erzeugung von PDF-Dateien wird eine PHP-Bibliothek als WebService angeboten, die im Wesentlichen HTML samt CSS in ein PDF/A-Dokument umwandelt.

Funktionsumfang

  • Umwandeln von HTML
  • Umwandeln von CSS
  • Erzeugung von PDF/A-Dokumente
  • Aufbringung eines nicht manipulierbaren Zeitstempels
  • Anpassung an vordefinierbare Vorlagen
  • Bereitstellung von individuellen WebServices

Technik

  • Pfad: https://pdfe.iabg.de
  • HTTP und HTTPS werden unterstützt
  • Produktive PDF-WebServices befinden sich im Verzeichnis /var/www/html/serv.pdfe/
  • Testweise PDF-WebServices befinden sich im Verzeichnis /var/www/html/serv.pdfe.dev/
  • Zugangsgeschützte PDF-WebServices befinden sich im Verzeichnis /var/www/html/serv.pdfe.Name (Name ist der Platzhalter)
    Die Absicherung wird mittels Basic-Auth vorgenommen, wobei eine AD-Integration NICHT unterstützt wird.

Doccen

Warum mit Doccen dokumentieren

Mit Hilfe von Doccen wird eine API-nahe Dokumentation direkt aus dem Quellcode erzeugt. Vordergründig wird dies gemacht, um überhaupt eine Dokumentation zu haben, da diese sonst meist sehr aufwändig ist oder dem Stand der Dinge zeitlich nachläuft. Da man den Sourcecode ohnehin dokumentieren muss, wird durch eine spezielle Notation mit diesem Aufwand gleich eine besser lesbare Dokumentation erzeugt.

Doccen selbst wurde eigens für die Schulungsvideosserien von Share2Brain.com entwickelt und mehr Infos, sowie Schulungsvideos, finden sie unter doccen.com

Info zu "AutoDoc"

AutoDoc ist das Kernmodul zur Erstellung von Dokumentationen bei Share2Brain.com, wird aber zusammen mit zwei Erweiterungsmodulen angeboten, die einen Worklfow weiter unterstützen können. "AutoDoc" ist die zentrale Komponente, um die der Prozess zu einer Dokumentationserzeugung aufgebaut wurde. Wichtig Kernziele sind:

  • schneller
  • einfacher
  • integriert
  • aktueller und besser

Info zu "Composer"

Aus einzelnen Programmbausteinen, kann man mit der Erweiterung "Composer" einfach Pakete zusammenstellen. Dieser Schritt ist Optional.
DIESE ERWEITERUNG BEFINDET SICH IN EINER FRÜHEN ENTWICKLUNGSPAHSE!

Info zu "Grabber"

Sehr hilfreich für die Beschleunigung der Dokumentationserzeugung ist der "Grabber", der die Dateien aus Ihrem Entwicklungsrepository zusammenzieht. Kopieren sie einfach den letzten Stand ihrer Entwicklungen in das SourceS-Verzeichnis und bestimmen mit dem Grabber, in welche Dokumentationen welche Dateien mit einbezogen werden sollen.
DIESE ERWEITERUNG BEFINDET SICH IN EINER FRÜHEN ENTWICKLUNGSPAHSE!

Info zu "Workflow"

Um die einzelnen Komponenten von Composer, Grabber und AutoDoc in eine automatisierte Abfolge zu bringen, kommt Workflow zum Einsatz. Hier werden entsprechend die einzeln definierten Tasks ausgeführt.

doccen.com vs Doccen?

Doccen ist ein autonomes System und KEIN Cloud-Service. In den Schulungsvideos wird gezeigt, wie leicht es ist, Doccen lokal oder auf dem eigenen WebServer zu betreiben. doccen.com ist die WebSite, auf der derzeit Videoschulungen und Updates angeboten werden.
